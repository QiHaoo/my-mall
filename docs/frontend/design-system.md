## 二、主题定制

### 2.1 品牌色

```scss
// Element Plus CSS 变量覆盖
:root {
  --el-color-primary: #409eff;       // 主色（品牌蓝）
  --el-color-success: #67c23a;
  --el-color-warning: #e6a23c;
  --el-color-danger: #f56c6c;
  --el-color-info: #909399;
}
```

> 生产环境主题色通过 SCSS 变量覆盖，不使用运行时动态主题（避免 FOUC）。

### 2.2 间距规范

| 用途 | 值 |
|------|---|
| 组件内间距 | 8px / 12px |
| 卡片间距 | 16px / 24px |
| 页面边距 | 24px |

### 2.3 字号规范

| 用途 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 20px | 600 |
| 卡片标题 | 16px | 500 |
| 正文 | 14px | 400 |
| 辅助文字 | 12px | 400 |

## 三、通用布局

### 3.1 后台管理（mall-admin）

```
┌──────────────────────────────────────────┐
│  顶部导航栏（Logo + 面包屑 + 用户菜单）     │
├────────┬─────────────────────────────────┤
│        │                                 │
│ 侧边栏  │       内容区域                   │
│ 菜单    │   （路由出口 <router-view>）      │
│        │                                 │
├────────┴─────────────────────────────────┤
│  底部（版权信息，可选）                      │
└──────────────────────────────────────────┘
```

- 侧边栏：`el-menu` + `el-sub-menu`，支持折叠
- 面包屑：`el-breadcrumb`，从路由 meta 自动生成
- 内容区：`<router-view>` + `<transition>` 过渡动画

## 四、通用组件规范

### 4.1 表格页（CRUD 标准模式）

每个功能域的列表页统一结构：

```
┌─ 搜索栏（el-form inline + 查询/重置按钮）─────┐
├─ 工具栏（新增按钮 + 批量操作下拉）────────────┤
├─ 数据表格（el-table + 分页 el-pagination）───┤
└─ 弹窗表单（el-dialog + el-form）─────────────┘
```

**约定**：
- 搜索栏字段 ≤ 5 个时一行排列，> 5 个折叠展开
- 表格操作列固定在右侧，按钮文字精简（编辑/删除）
- 分页默认 `page-size=10`，可切换 10/20/50
- 弹窗表单宽度：新增/编辑 `500px`，详情 `700px`

### 4.2 表单页

- 必填字段用 `*` 标记（Element Plus 默认行为）
- 校验规则：前端校验 + 后端校验双层，后端返回的错误用 `el-form` 的 `validateField` 回显
- 提交按钮 loading 状态防重复提交

### 4.3 树形组件

三级分类、地区等树形数据统一用 `el-tree`：
- 支持展开/折叠
- 支持拖拽排序（`draggable` 属性）
- 右键菜单（新增子项/编辑/删除）

## 五、交互规范

| 场景 | 交互方式 |
|------|---------|
| 删除操作 | `el-message-box` 二次确认，危险操作用 `type: 'warning'` |
| 操作反馈 | 成功 `el-message` success，失败 `el-message` error |
| 加载状态 | 表格用 `v-loading`，按钮用 `:loading` |
| 空数据 | `el-empty` 插槽，支持自定义操作按钮 |
| 网络错误 | 全局拦截器统一弹 `el-notification` |

## 六、响应式断点

| 设备 | 断点 | 布局调整 |
|------|------|---------|
| 桌面 | ≥ 1200px | 侧边栏 + 内容区 |
| 平板 | 768px ~ 1199px | 侧边栏折叠，内容区全宽 |
| 手机 | < 768px | 底部 Tab 导航，单列布局 |

> 后台管理（mall-admin）以桌面端为主，不强制适配移动端。商城前台需适配三端。

## 七、Figma 设计交付规范

### 7.1 命名规范

| 层级 | 命名示例 |
|------|---------|
| Page | `商品中心 / 分类管理` |
| Frame | `Desktop - 列表页` / `Desktop - 新增弹窗` |
| 组件 | `CategoryTree` / `BrandForm` |

### 7.2 交付物

- 高保真设计稿（含交互标注）
- 组件切图（SVG / PNG @2x）
- Dev Mode 标注（间距、颜色、字号）

### 7.3 与前端组件的对应

Figma 中的组件应与 Vue 组件一一对应，命名保持一致：
- Figma `CategoryTree` → Vue `CategoryTree.vue`
- Figma `BrandForm` → Vue `BrandForm.vue`

### 7.4 AI Agent 辅助生成设计稿

AI Agent（Trae + figma-console-mcp）可直接在 Figma 画布上生成、修改设计稿。完整的方案选型、环境配置、连接验证与生成流程见 [figma-mcp-guide.md](figma-mcp-guide.md)。

**关键能力**：
- 通过 `figma_execute` 执行 Figma Plugin API 代码，创建 Frame/Text/Rectangle 等节点
- 通过 `figma_take_screenshot` 截图验证，形成"生成 → 验证 → 迭代"闭环
- 支持设计变量管理、组件实例化等

**使用场景**：
- 功能域文档"前端设计"章节的页面布局，可由 AI 直接生成 Figma 稿
- 设计系统变量批量创建
- 快速出原型验证布局
