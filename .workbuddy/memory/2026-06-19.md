## WSL2 systemd 未启用问题
- 用户执行 `sudo systemctl start docker` 报 `System has not been booted with systemd as init system (PID 1)`
- 原因：WSL2 默认不启用 systemd，而 Docker 的 systemctl 命令依赖它
- 解决：写 `/etc/wsl.conf` 设 `[boot] systemd=true`，PowerShell 执行 `wsl --shutdown` 重启 WSL2，`ps -p 1` 验证输出 systemd
- 已更新 dev-environment-setup.md：Docker 安装步骤中补充 systemd 启用说明、FAQ 新增对应条目
- 教训：WSL2 环境搭建文档应在装任何依赖 systemd 的服务前，先启用 systemd
