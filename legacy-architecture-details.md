# 谷粒商城 — 微服务架构图内容整理

> 来源：原架构图「谷粒商城-微服务架构图」（约 2020 年前技术栈）

---

## 一、部署区域划分

架构图整体划分为两大区域：

| 区域 | 说明 |
|------|------|
| **外网部署** | 面向公网用户的接入层，包括 DNS、CDN、WAF、Nginx |
| **内网部署** | 微服务核心、数据层、可观测性、CI/CD 等内部系统 |

## 二、流量入口层（外网部署）

```
用户设备（手机 / 电脑 / 笔记本）
       │
       ▼
   gulishop.com（域名）
       │
       ▼
   DNS 域名解析
       │
       ▼
   WAF（Web 应用防火墙）
       │
       ▼
   CDN（内容分发网络）
       │

       │
       ▼

```

| 组件 | 数量 | 功能 |
|------|------|------|
| WAF | — | Web 应用防火墙，防御 SQL 注入、XSS 等攻击 |
| CDN | — | 静态资源加速，减轻源站压力 |
| Nginx | 集群 | 四层/七层负载均衡，反向代理，SSL 终止 |
| Spring Cloud Gateway | 3 实例 | 动态路由、鉴权、限流、跨域处理，基于 Webflux 响应式模型 |

## 三、微服务核心层（内网部署）

### 3.1 注册与配置中心

| 组件 | 功能 | 说明 |
|------|------|------|
| **Nacos（注册中心）** | 服务注册与发现 | 各微服务启动时注册到 Nacos，服务间通过 Nacos 发现目标地址 |
| **Nacos（配置中心）** | 集中配置管理 | 支持动态配置推送、配置隔离、配置回滚，各服务无需本地存储配置 |

两者通过数据库持久化元数据，支持双向通信（服务上报 + 配置推送）。

### 3.2 安全认证

| 组件 | 功能 |
|------|------|
| **OAuth2.0 认证中心** | 统一颁发和验证 Token，支持授权码模式、客户端凭证模式等 |
| **Spring Security** | 各微服务内的权限校验、资源保护、安全过滤 |

网关层统一做 Token 校验，内部服务通过 Spring Security 做细粒度权限控制。

### 3.3 服务间通信与治理

| 组件 | 功能 | 说明 |
|------|------|------|
| **Ribbon** | 客户端负载均衡 | 从 Nacos 获取服务实例列表，按策略（轮询/随机/权重）选择目标节点 |
| **Feign** | 声明式 HTTP 客户端 | 通过接口注解方式发起服务间调用，底层集成 Ribbon + Hystrix |
| **Sentinel** | 流控与熔断 | 接口级别限流、熔断降级、系统保护，配有 Sentinel Dashboard 控制台 |

### 3.4 业务微服务集群

架构图中有 **4 组业务服务集群**，每组均为多个 Spring Boot 实例，通过 Feign 相互调用，通过 Ribbon 负载均衡，受 Sentinel 保护。

> 图中标注：每个服务集群都包含 SpringBoot + Feign 的组合，支持水平扩展。

横切关注点（绿色标签）：
- 动态配置（Nacos 推送）
- 配置隔离（环境隔离：dev / test / prod）
- 配置回滚（Nacos 版本回滚）
- 熔断降级（Sentinel）
- 链路追踪（Sleuth）
- 告警（Alertmanager）

## 四、可观测性层

### 4.1 链路追踪

```
微服务调用链
       │
       ▼
  Sleuth（注入 Trace ID / Span ID）
       │
       ▼
  Zipkin Server（收集与可视化调用链）
       │
       ▼
  Prometheus（将追踪延迟指标采集存储）
```

| 组件 | 功能 |
|------|------|
| **Sleuth** | 在请求头中注入 TraceID 和 SpanID，贯穿整个调用链 |
| **Zipkin** | 收集各服务上报的 Span 数据，可视化展示调用链路与耗时 |

### 4.2 监控与告警

```
各微服务 Metrics 上报
       │
       ▼
  Prometheus（拉取/存储指标）
       │
       ├──► Grafana（可视化大盘展示）
       │
       └──► Alertmanager（告警路由）
                │
                ├──► 邮件通知
                └──► 短信通知
```

| 组件 | 功能 |
|------|------|
| **Prometheus** | 拉取各服务的 Metrics（QPS、延迟、错误率、JVM 指标等），支持 PromQL 查询 |
| **Grafana** | 将 Prometheus 数据渲染为可视化大盘，支持多数据源聚合 |
| **Alertmanager** | 根据 Prometheus 告警规则触发通知，支持邮件、短信、钉钉等渠道 |

### 4.3 日志系统（ELK Stack）

```
各微服务日志输出
       │
       ▼
  LogStash（采集 + 过滤 + 格式化）
       │
       ▼
  Elasticsearch（存储 + 索引）
       │
       ▼
  Kibana（可视化查询与分析）
       │
       ├──► 日志存储（历史日志归档查询）
       └──► 文书检索（业务日志内容检索）
```

| 组件 | 功能 |
|------|------|
| **LogStash** | 从各服务采集日志，进行过滤、格式化、富化处理后写入 ES |
| **Elasticsearch** | 分布式存储日志数据，提供全文检索能力 |
| **Kibana** | Web UI 查询日志，构建 Dashboard，支持日志趋势分析 |

## 五、数据存储层

### 5.1 分布式缓存

| 组件 | 模式 | 功能 |
|------|------|------|
| **Redis** | Sentinel（哨兵）+ Shard（分片） | 3 节点集群，提供分布式缓存，支持高可用自动故障转移 |

主要用途：
- 热点数据缓存（商品信息、用户 Session）
- 分布式锁
- 排行榜（ZSet）
- 购物车（Hash）
- 限流计数（INCR）

### 5.2 持久化数据库

| 组件 | 模式 | 功能 |
|------|------|------|
| **MySQL** | 主从复制（Master-Slave） | 3 节点，主库负责写，从库负责读，数据持久化 |

架构图中「持久化」标签直连 MySQL 集群，支持读写分离。

### 5.3 消息队列

| 组件 | 模式 | 功能 |
|------|------|------|
| **RabbitMQ** | Mirror（镜像集群） | 3 节点，消息持久化，异步解耦业务 |

主要用途：
- 订单创建后异步扣减库存
- 发送通知消息（邮件/短信）
- 流量削峰（秒杀场景）
- 延迟队列（订单超时取消）

### 5.4 全文检索

| 组件 | 模式 | 功能 |
|------|------|------|
| **Elasticsearch** | Shard（分片） | 4 节点集群，提供商品全文检索与聚合分析 |

主要用途：
- 商品关键词搜索
- 分类/品牌/价格区间聚合筛选
- 搜索建议与高亮显示

### 5.5 对象存储

| 组件 | 功能 |
|------|------|
| **OSS** | 存储商品图片、视频、文件等静态资源，配合 CDN 加速访问 |

## 七、CI/CD 流水线

```
开发者
  │
  ▼
GitHub（代码托管）
  │
  ▼
Jenkins Pipeline（CI 自动化）
  │  ├── 代码拉取
  │  ├── 编译构建（Maven/Gradle）
  │  ├── 单元测试
  │  └── 镜像构建（Dockerfile）
  │
  ▼
Docker 镜像仓库（存储构建产物）
  │
  ▼
Kubernetes API（容器编排）
  │
  ▼
K8s 集群（UAT / PROD 环境）
  │  └── 8 个容器实例（滚动更新 / 弹性扩缩）
  │
  ▼
OP 运维团队（监控 + 管理 K8s 集群）
```

| 阶段 | 组件 | 说明 |
|------|------|------|
| 代码托管 | **GitHub** | 开发提交代码，触发 CI 流程 |
| CI 自动化 | **Jenkins Pipeline** | 拉取代码 → 编译 → 测试 → 构建镜像，全流程自动化 |
| 镜像存储 | **Docker 镜像仓库** | 存储每次构建的 Docker 镜像，带版本标签 |
| 容器编排 | **Kubernetes (K8s)** | 管理容器生命周期，支持滚动更新、回滚、弹性扩缩 |
| 环境 | **UAT / PROD** | 测试环境与生产环境，均在 K8s 内管理 |
| 运维 | **OP 运维团队** | 通过 K8s API / Dashboard 管理集群 |
